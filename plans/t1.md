TASK TITLE
Create Monorepo Structure for License Plate Recognition Desktop App

SUMMARY
Thiết lập cấu trúc monorepo với 3 workspace chính: /src-tauri (Rust backend), /src (React frontend), /ai-service (Python AI service) làm nền tảng cho toàn bộ dự án LPR.

BACKGROUND / CONTEXT
- Đây là task đầu tiên của Phase 1 MVP, mọi task khác đều phụ thuộc vào cấu trúc repo này
- Dự án LPR sử dụng kiến trúc 3 lớp: Tauri v2 (Rust) + React/Vite (frontend) + FastAPI (Python AI inference)
- Cần monorepo structure rõ ràng để team có thể làm việc song song trên các module khác nhau
- Không có repo structure chuẩn → integration giữa các component sẽ khó khăn và tốn thời gian

SCOPE OF WORK
1. Tạo root project directory với file cấu hình chung (.gitignore, README.md, .editorconfig)
2. Tạo /src-tauri directory — Rust/Tauri backend
   - Cargo.toml với Tauri v2 dependencies
   - src/main.rs — entry point
   - src/lib.rs — shared library
   - tauri.conf.json — Tauri configuration
   - icons/ — app icons placeholder
3. Tạo /src directory — React frontend
   - package.json với React + Vite + TypeScript dependencies
   - vite.config.ts — Vite config tích hợp Tauri
   - tsconfig.json — TypeScript config
   - index.html — entry HTML
   - src/App.tsx — root component placeholder
   - src/main.tsx — React entry point
4. Tạo /ai-service directory — Python AI service
   - pyproject.toml hoặc requirements.txt với FastAPI, uvicorn, opencv-python
   - main.py — FastAPI entry point placeholder
   - models/ — directory cho YOLO + OCR model files
   - .python-version — specify Python version
5. Tạo shared config files ở root
   - .gitignore (Rust, Node, Python, IDE files, model weights)
   - README.md với project overview và setup instructions
6. Verify toàn bộ structure bằng tree command và commit initial structure

ACCEPTANCE CRITERIA
- [ ] Repo có 3 top-level directories: src-tauri/, src/, ai-service/
- [ ] .gitignore cover tất cả 3 stack (Rust target/, node_modules/, __pycache__/, .venv/, *.onnx, *.pt)
- [ ] README.md mô tả project structure, prerequisites, và basic setup steps
- [ ] Mỗi sub-project có config file cơ bản (Cargo.toml, package.json, pyproject.toml/requirements.txt)
- [ ] Directory structure clean, không chứa generated files hay dependencies
- [ ] Git history clean với 1 initial commit chứa toàn bộ scaffold
